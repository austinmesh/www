    <h1>Understanding LoRa Modulation through Comparison</h1>
    <p>Written by <a href="https://k2xap.radio">Alex, K2XAP</a> of <a href="https://nyme.sh" rel="nofollow noopener" target="_blank">nyme.sh</a></p>
    <p>LoRa packets come in a wide variety of shapes and sizes, depending on their parameters. These shapes bring with them pros and cons. One of the biggest impacts is the airtime. Meshes have to <a href="https://meshtastic.org/blog/why-your-mesh-should-switch-from-longfast/">choose</a> between range, noise tolerance, and time on the air, finding the right balance of link reliability and congestion for their density and location.</p>
  </header>

  <p>At the default settings, a Meshtastic packet can take literally a <strong>whole second</strong> to transmit. This gives it great range and resilience, but a mesh can quickly become congested as soon as it exceeds the number of contacts a typical device can hold. Faster settings sacrifice range for much lower airtime, and can use infrastructure to compensate for lower link reliability with retries of packets. Each step faster in Meshtastic presets is a roughly 45% reduction in airtime.</p>
  <p>The bandwidth also changes the exposure to noise. Wider bandwidth gives LoRa a much quicker transmission speed and reduces risk of "vertical" collisions in the temporal dimension, but it increases the chance of "horizontal" collisions in the spectral dimension—in addition to increased thermal noise exposure.</p>

  <h2 id="demonstration">Demonstration</h2>
  <p>These steps will reproduce the above demonstration. This uses MeshCore because it can change radio settings without a device reboot, but the output is equivalent to any LoRa transmission.</p>
  <ol>
    <li>
      <p>Tune an SDR to the desired frequency. The script defaults to 912.4 MHz.</p>
    </li>
    <li>
      <p><a href="https://flasher.meshcore.io/" rel="nofollow noopener" target="_blank">Flash</a> a node as a USB MeshCore companion.</p>
    </li>
    <li>
      <p>Install the <a href="https://github.com/meshcore-dev/meshcore_py" rel="nofollow noopener" target="_blank">meshcore python library</a>: <code>pip install meshcore</code>.</p>
    </li>
    <li>
      <p>Run the script (set the <code>INTERFACE</code> to the connected device):</p>
    </li>
  </ol>

```python
import asyncio
from meshcore import MeshCore, EventType

INTERFACE = "/dev/tty.usbmodem1301"  # Set this to your local device
FREQUENCY = 912.4 # MHz, Change this to the desired frequency
PRESETS = [
    (250,11,5),   # MT LongFast
    (250,10,5),   # MT MediumSlow
    (250,9,5),    # MT MediumFast
    (250,8,5),    # MT ShortSlow
    (250,7,5),    # MT ShortFast
    (125,8,5),    # MeshOregon
    (62,7,5),     # MC Narrow
    (62,9,5),     # MC Narrow Long
    (500,11,8),   # MT LongTurbo
    (500,7,5),    # MT ShortTurbo
]

async def main():
    meshcore = await MeshCore.create_serial(INTERFACE)
    await meshcore.commands.set_tx_power(1) # Be a good RF neighbor
    await asyncio.sleep(1)
    for (bw,sf,cr) in PRESETS:
        await meshcore.commands.set_radio(FREQUENCY, bw, sf, cr)
        print((bw, sf, cr))
        result = await meshcore.commands.send_chan_msg(0, "The quick brown fox jumped over the lazy dog")
        print(result)
        await asyncio.sleep(1)
    await meshcore.disconnect()

asyncio.run(main())
```
